**Correctness checks.** The invertibility check computed z = f⁻¹(x) then x̂ = f(z) for every training point, yielding a maximum absolute reconstruction error of ~1.8 × 10⁻¹⁵ — within float64 machine precision. Per-sample errors show discrete banding at small multiples of machine epsilon, reflecting quantised IEEE 754 round-off across eight coupling layers — purely precision artefacts, not bugs. The log-determinant check compared the analytic log|det J| to a central-difference estimate (ε = 10⁻⁴) at the first training point, giving an absolute error of ~1.4 × 10⁻⁹, well below the O(ε²) ≈ 10⁻⁸ truncation error, confirming the analytic log-det is correct.

**Training choices and diagnostics.** Training followed a six-step recipe. On the tiny subset (first 128 rows), three configurations were compared: without scheduling or clipping the flow was unstable with exploding gradients; cosine scheduling alone produced extreme overfitting with erratic dynamics; cosine scheduling combined with aggressive gradient clipping (norm = 0.1) gave stable convergence. This confirmed the pipeline was sound and the model had sufficient capacity. Scaling to the full 800-sample dataset without regularisation showed textbook overfitting — validation loss diverged sharply after ~500 steps while training loss continued to fall. A regularisation ablation found that early stopping (patience = 1000) had the largest impact, while weight decay (1 × 10⁻⁴) slightly degraded performance, likely because L2 penalisation restricts the model's ability to fit the bimodal structure on such a small dataset. A hyperparameter grid search initially selected lr = 3 × 10⁻³, but this proved unstable — the model was getting lucky with early stopping rather than converging reliably. The final model used Adam with lr = 3 × 10⁻⁴, cosine LR schedule, He initialisation, gradient clipping (norm = 1.0), hidden = 128, and n_layers = 8, achieving a 42% improvement over a full-covariance Gaussian baseline on the test set.

**Surgery map.** The shear map g_α transforms x as y₁ = x₁ + αx₂, y₂ = x₂. Its Jacobian is upper-triangular with unit diagonal, so |det J| = 1 and log|det J| = 0 — the map is volume-preserving. Appending g_α after the trained flow creates a new sampling map f_α(z) = g_α(f₀(z)) that horizontally distorts the learned density by an amount proportional to α times each point's vertical coordinate. At α = 0 the original two-moon density is recovered; increasing |α| progressively shears the moons, tilting them while preserving total probability mass.

**FLOP count assumptions.** (1) Masking operations (applying binary masks to select transformed dimensions) are not counted. (2) The inverse affine transforms (exp, subtraction, multiplication) are evaluated densely across all D dimensions before masking, rather than on only the D/2 active dimensions. This matches the specified architecture — weight matrices have D input columns regardless of masking. (3) Transcendental functions are costed at 10 FLOPs each (exp, tanh) following the provided table. (4) Memory access, indexing, data movement, and Python overhead are ignored — only arithmetic operations are counted. (5) All per-sample operations scale linearly with batch size B, assuming no batch-level overhead.
